# Фича 08: Drawer манёвров в GameMode

## Problem Statement

Сейчас в `GameMode` уже есть удобный универсальный блок `Бросок кубиков / Инициатива`, но боевые манёвры живут отдельно в длинной текстовой `Справке по бою` ниже по экрану.

Из-за этого в бою возникает лишнее трение:

- игроку нужно каждый раз перечитывать правило манёвра,
- затем вручную переводить его в `атрибут + способность + сложность`,
- затем вручную настраивать роллер,
- быстрый боевой выбор не отделён от полной справки,
- текущие манёвры захардкожены прямо внутри `GameMode.tsx`, поэтому любой новый быстрый слой рискует разъехаться с уже существующей справкой.

В результате интерфейс либо перегружен текстом, либо не помогает быстро выбрать типовое действие.

## Goal

Добавить в `GameMode` рядом с блоком `Бросок кубиков / Инициатива` кнопку `⚔ Манёвры`, которая открывает правый `drawer` с вкладками:

- `Атака`
- `Защита`
- `Дистанция`
- `Особое`

Во вкладках должны быть карточки манёвров с короткой сводкой:

- `пул`,
- `сложность`,
- `урон`,
- `ограничение`.

У каждой карточки две кнопки:

- `Подставить в бросок`
- `Подробнее`

`Подставить в бросок` включает явный пресет манёвра в уже существующем роллере.
`Подробнее` показывает расширенный текст этого манёвра внутри того же `drawer`.

Текущая текстовая `Справка по бою` на первом этапе остаётся, но должна быть синхронизирована с новым каталогом манёвров и пополнена новыми манёврами.

## Non-Goals (для этой фичи)

- Автоматизация полного боевого пайплайна `попадание -> защита -> урон -> soak`.
- Отдельный полуавтоматический бросок урона.
- Изменения `CombatPage`.
- Удаление текущей текстовой `Справки по бою`.
- Введение структурированной модели оружия, магазина, инвентаря или `accuracy` оружия в данных персонажа.
- Backend/API/DTO-изменения.
- Универсальный design-system `drawer/tabs`, используемый по всему приложению.
- Введение полноценной клиентской test-инфраструктуры, которой сейчас в репозитории нет.

## Текущие ограничения кода

- `apps/client/src/components/GameMode.tsx` уже содержит роллер, инициативу, лог, здоровье, traits и боевую справку; дальнейшее наращивание инлайн-данных внутри этого файла ухудшит поддержку.
- Текущее поведение роллера строится вокруг `selectedAttributeKey`, `selectedAbilityKey`, `diceDifficulty` и `diceCount`.
- При выборе `атрибута + способности` пул уже автоматически пересчитывается через текущий `useEffect`.
- `CharacterDto.equipment` — это просто текстовое поле, поэтому `Точность` в рамках этой фичи должна трактоваться только как бонус самого манёвра, а не как свойство конкретного оружия.
- Готового переиспользуемого `drawer` или `tabs` на клиенте сейчас нет.
- При конфликте между новой внешней табличкой и уже существующей `Справкой по бою` источником истины считается текущая справка.
- Новые согласованные манёвры, которых сейчас нет в справке, нужно добавить и в `drawer`, и в старую текстовую справку, чтобы не получить две расходящиеся базы правил.

## Зафиксированные продуктовые решения

- Область фичи: только `GameMode`.
- Текущая текстовая `Справка по бою` в `v1` сохраняется.
- Контейнер: всегда правый `drawer` на всех брейкпоинтах.
- На узких экранах `drawer` может становиться почти на всю ширину, но остаётся именно `drawer`, а не `bottom sheet`.
- `Подставить в бросок` никогда не делает авто-бросок.
- Первая версия работает как явный пресет манёвра поверх текущего универсального роллера:
  - карточка выбирает манёвр,
  - манёвр подставляет `атрибут`,
  - манёвр подставляет `способность`,
  - манёвр подставляет итоговую `сложность`,
  - манёвр добавляет свой бонус к пулу, если он у него есть.
- После применения манёвра роллер должен явно показывать, какой манёвр сейчас активен.
- Если пользователь вручную меняет `атрибут` или `способность`, активный пресет манёвра сбрасывается.
- Если пользователь меняет только `сложность` или `кубы`, активный пресет манёвра не сбрасывается.
- После нажатия `Подставить в бросок` `drawer` остаётся открытым и визуально подсвечивает активную карточку.
- `Подробнее` открывает расширенный текст внутри того же `drawer`, а не через отдельный modal/popover и не через прокрутку к старой справке.
- Для процедурных или модификаторных карточек без одного основного броска кнопка `Подставить в бросок` остаётся видимой, но disabled.
- Дополнительные пометки `(А)`, `(Д)`, `(С)`, `(У)` допустимы в деталях/подсказках и легенде, но не должны перегружать основную карточку.

## Нормализация названий в v1

- Карточка `Выстрел` соответствует текущему пункту справки `Одиночный выстрел`.
- Текущий пункт `Подсечка` разделяется на два отдельных манёвра:
  - `Подножка`
  - `Подсечка оружием`
- Текущий пункт `Разоружение` рекомендуется нормализовать в `Разоружение (оружием)`, чтобы парой к нему добавить новый манёвр `Разоружение (голыми руками)`.
- Текущее название `Удар рукой` сохраняется как основное; если нужно, слово `кулаком` можно использовать только в detail-тексте.

## Рекомендуемая форма реализации

Чтобы не раздувать `GameMode.tsx` ещё сильнее, фичу лучше разложить на три локальных файла:

- `apps/client/src/components/GameMode.tsx`
  - orchestration,
  - состояние роллера,
  - состояние активного манёвра,
  - apply/reset-логика.
- `apps/client/src/components/GameModeManeuversDrawer.tsx`
  - UI `drawer`,
  - вкладки,
  - карточки,
  - detail-область,
  - disabled/apply state.
- `apps/client/src/components/gameModeCombatManeuvers.ts`
  - единый каталог манёвров,
  - типы карточек,
  - пресеты,
  - detail-тексты,
  - группировка для legacy guide.

Это прагматичнее, чем тянуть feature в `shared`, потому что пока она специфична именно для `GameMode`.

## Целевая клиентская модель состояния

Новые состояния в `GameMode`:

- `maneuversOpen: boolean`
- `activeManeuverTab: "attack" | "defense" | "ranged" | "special"`
- `expandedManeuverId: string | null`
- `activeManeuverId: string | null`

Существующие состояния, которые будут переиспользованы:

- `selectedAttributeKey`
- `selectedAbilityKey`
- `diceDifficulty`
- `diceCount`

Ключевое производное поведение:

- базовый пул = `атрибут + способность + woundMod`,
- итоговый предложенный пул при активном манёвре = `базовый пул + бонус манёвра`,
- ручная правка `diceCount` разрешена и не обязана сбрасывать манёвр,
- применение другого манёвра заменяет предыдущий пресет,
- если пользователь вручную выбрал другую пару `атрибут + способность`, активный манёвр снимается.

## Единый каталог манёвров

У каталога должен быть один источник истины и для `drawer`, и для legacy-секций манёвров в `Справке по бою`.

Рекомендуемые поля карточки:

- `id`
- `tab`
- `title`
- `summary.pool`
- `summary.difficulty`
- `summary.damage`
- `summary.limitation`
- `applyMode: "enabled" | "disabled"`
- `preset?`
  - `attributeKey`
  - `abilityKey`
  - `appliedDifficulty`
  - `diceBonus`
- `detailText`
- `detailFlags`
- `guideSection`

Важно:

- `Бой: фазы` и `Попадание, защита, урон, soak` могут остаться отдельными explanatory-блоками.
- Но манёвренные разделы старой справки больше не должны жить отдельным захардкоженным текстом, если это можно избежать.
- Предпочтительный путь: рендерить манёвренные секции старой справки из того же каталога, что и карточки `drawer`.

## Легенда для detail/help текста

- `(А)` — манёвр наносит агравированные повреждения.
- `(Д)` — манёвр длится, пока персонаж набирает успехи.
- `(С)` — манёвр сбивает с ног.
- `(У)` — манёвр уменьшает количество успехов у атакующего.

Эта легенда должна появляться либо в detail-области `drawer`, либо в его footer/help-блоке, но не должна перегружать компактные карточки.

## Пошаговый план

### Step 1. Вынести единый каталог манёвров и зафиксировать source of truth

1. Создать отдельный pure-модуль каталога манёвров рядом с `GameMode`.
2. Описать в нём:
   - вкладку,
   - заголовок,
   - summary-поля,
   - detail-текст,
   - применимость `Подставить`,
   - пресет роллера (`attributeKey`, `abilityKey`, `appliedDifficulty`, `diceBonus`).
3. Перенести в каталог текущие манёвры из `GameMode.tsx`, сохранив именно ту механику, которая уже зафиксирована в существующей справке.
4. Добавить новые согласованные манёвры:
   - `Удар когтями`
   - `Разоружение (голыми руками)`
   - `Бросок оружия`
   - `Быстрая перезарядка (револьвер/дробовик)`
5. Расщепить старую общую запись `Подсечка` на:
   - `Подножка`
   - `Подсечка оружием`
6. Нормализовать именование карточек и guide-пунктов, чтобы один и тот же манёвр не существовал под разными несовместимыми названиями.

Done criteria:

- Есть один клиентский источник истины для всех манёвров.
- Каталог достаточно полон, чтобы из него рендерить и карточки `drawer`, и legacy-секции манёвров.

### Step 2. Собрать локальный `drawer`-компонент для манёвров

1. Добавить кнопку `⚔ Манёвры` рядом с текущим блоком `Бросок кубиков / Инициатива`.

## Риски и места особого внимания

- Самый большой риск — дублирование правил между карточками и старой справкой. Поэтому единый каталог манёвров здесь важнее, чем сам `drawer`.
- `GameMode.tsx` уже большой; если не вынести `drawer` и каталог в отдельные файлы, компонент быстро станет трудно поддерживать.
- Бонусы `+2` / `+10` для дистанционных манёвров должны трактоваться именно как бонус манёвра к пулу, а не как скрытая логика оружия.
- `details-only` карточки должны оставаться полезными и не выглядеть сломанными только из-за disabled `Подставить`.
- Нужно внимательно не сломать существующий auto-pool flow от клика по traits.
